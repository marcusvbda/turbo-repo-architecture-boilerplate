# Monorepo Boilerplate

## Goal

Create a Turborepo monorepo that runs React Native (Expo), Next.js, and NestJS with a single command, sharing node_modules, types, and configurations across all apps. Local Supabase as the database.

## Shared Packages

### `packages/tsconfig`

Base TypeScript configs extended by each app:

```
packages/tsconfig/
‚îú‚îÄ‚îÄ base.json          # strict mode, paths, moduleResolution
‚îú‚îÄ‚îÄ nextjs.json        # extends base + next plugin
‚îú‚îÄ‚îÄ react-native.json  # extends base + RN types
‚îî‚îÄ‚îÄ nestjs.json        # extends base + decorators, emitDecoratorMetadata
```

### `packages/eslint-config`

Shared ESLint rules across all apps. Each app creates an `.eslintrc.js` that extends the workspace config.

### `packages/types`

Single source of truth for TypeScript types:

```
packages/types/src/
‚îú‚îÄ‚îÄ supabase.ts   # auto-generated via `pnpm db:types`
‚îî‚îÄ‚îÄ index.ts      # re-exports everything
```

Usage in apps:

```ts

```

## Turborepo Pipeline (`turbo.json`)

```json
{
  "tasks": {
    "dev": {
      "persistent": true,
      "cache": false,
      "dependsOn": ["^build"]
    },
    "build": {
      "outputs": [".next/**", "dist/**"],
      "dependsOn": ["^build"]
    },
    "lint": {},
    "typecheck": {
      "dependsOn": ["^build"]
    }
  }
}
```

- `dev` is persistent (long-running process) and skips cache
- `build` respects package dependencies (`^build` = packages build first)
- `lint` and `typecheck` run in parallel across all workspaces

```json
  "private": true,
  "scripts": {

    "typecheck": "turbo typecheck",
    "db:reset": "supabase db reset",
    "db:types": "supabase gen types typescript --local > packages/types/src/supabase.ts",
    "db:migrate": "supabase migration new"
  }

```

## App and Package Cross-References

Each app declares internal dependencies using `workspace:*`:

```json
{
  "name": "@repo/web",
  "dependencies": {
    "@repo/types": "workspace:*"
  },
  "devDependencies": {
    "@repo/tsconfig": "workspace:*",
    "@repo/eslint-config": "workspace:*"
  }
}
```

The `@repo` namespace is used across all workspaces. pnpm automatically hoists common node_modules to the root ‚Äî no duplication.

## Local Supabase

The Supabase CLI starts a full Docker stack: Postgres, Auth, Storage, and Realtime.

| Service | Local URL |
|---|---|
| API / REST | http://localhost:54321 |
| Studio (UI) | http://localhost:54323 |
| Postgres | postgresql://localhost:54322 |
| Auth | http://localhost:54321/auth/v1 |

Running `pnpm db:types` regenerates `packages/types/src/supabase.ts` whenever the schema changes, keeping all apps in sync automatically.

## Global Commands

| Command | Description |
|---|---|
| `pnpm dev` | Start Next.js + NestJS + Expo + local Supabase |
| `pnpm build` | Build all apps in dependency order |
| `pnpm lint` | Lint all apps with shared rules |
| `pnpm typecheck` | Typecheck all apps |
| `pnpm db:reset` | Recreate local database with migrations + seed |
| `pnpm db:types` | Sync database types to `@repo/types` |
| `pnpm db:migrate` | Create a new migration file |

## Decisions and Constraints

- **pnpm** as package manager ‚Äî best workspace support and efficient hoisting
- **`shamefully-hoist=true`** in `.npmrc` ‚Äî required for Metro bundler (React Native)
- **Supabase at root** (`/supabase`) ‚Äî accessible by any app, migrations versioned with the code
- **`@repo` namespace** ‚Äî consistent across all workspaces, no npm publishing required
- UI sharing between RN and Next.js **out of scope** for this phase ‚Äî only node_modules, configs, and types are shared
- All text in docs, variable names, file names, database table/column names, and CSS classes must be in **English**
