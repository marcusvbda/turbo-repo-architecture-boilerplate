# Monorepo Boilerplate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scaffold a Turborepo monorepo with Next.js 16, Expo SDK 56, NestJS 11, shared packages, and local Supabase that boots with a single `pnpm dev` command.

**Architecture:** Three apps (`web`, `mobile`, `api`) under `apps/`, three shared packages (`tsconfig`, `eslint-config`, `types`) under `packages/`, and Supabase at the root `/supabase`. Turborepo orchestrates tasks in parallel. pnpm workspaces hoists common node_modules to the root.

**Tech Stack:** pnpm 9, Turborepo 2, Next.js 16 (App Router), Expo SDK 56 + Expo Router 56, NestJS 11, Supabase CLI (local), TypeScript 5, React 19.2, React Native 0.85.

## Global Constraints

- Node.js 20 LTS
- pnpm 9.x — no npm or yarn
- `@repo/*` namespace across all workspaces
- `workspace:*` for all internal dependencies
- `shamefully-hoist=true` in `.npmrc` — required for Metro/React Native bundler
- TypeScript strict mode in all apps
- Supabase at `/supabase` at project root (local CLI — no cloud)
- All text in docs, variable names, file names, database table/column names, and CSS classes must be in **English**
